Анализ A/B-теста: Оценка влияния пуш-уведомлений на ключевые метрики B2B-клиентов

Этот проект представляет собой полный цикл анализа результатов A/B-тестирования, направленного на оценку эффективности нового маркетингового инструмента (пуш-уведомления) в сравнении со стандартным подходом (баннерная реклама).

Методология и этапы проекта
1. Подготовка и очистка данных
Источники: 3 таблицы (Заказы, Клиенты, Справочник регионов) из исходного Excel-файла.
Инструмент: Python (Pandas).
Критерии чистоты: Удалены строки, не имеющие критических идентификаторов (id_client, id_order, amt_payment).
Трансформация: Объединение таблиц (LEFT JOIN) и расчет агрегированных метрик (сумма платежей, число заказов, флаг оплаты) на уровне клиента.
2. Валидация и фильтрация
Этот этап критичен для обеспечения чистоты A/B-теста, что демонстрирует глубокое понимание методологии.
Проверка на чистоту ТТ: Проведена дополнительная фильтрация торговых точек (ТТ), где одна из групп (Тест или Контроль) имела нулевую сумму продаж, что могло бы исказить средний чек.
Результат фильтрации: Из 55,605 клиентов удалено около 9,800 клиентов, связанных с невалидными ТТ, что обеспечило надежность статистических выводов.
3. Статистический анализ
Метрики: Средняя сумма платежей на клиента и Конверсия (флаг совершения покупки).
Используемые тесты:
T-тест Уэлча (Welch's t-test): Для сравнения средних значений (подтверждение разницы в среднем чеке и конверсии).
Критерий Манна-Уитни: Непараметрический тест для сравнения распределений сумм платежей (используется как дополнительная проверка, так как распределение чеков обычно не нормальное).
Уровень значимости (α): 0.05.
4. Региональная сегментация
Проведен детальный анализ эффективности по крупным городам (Москва, Санкт-Петербург) и по каждой торговой точке отдельно для выявления локальных положительных, нейтральных или отрицательных эффектов.
